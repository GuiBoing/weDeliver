Desafio vaga Desenvolvedor Front-End UI/UX

Introdução

Este desafio tem como objetivo avaliar suas habilidades em desenvolvimento Front-End, design de interface de usuário e experiência do usuário (UI/UX). Você terá 72 horas para completar o desafio e enviar o resultado final para um repositório no GitHub.

Descrição do Projeto

Você deve criar uma nova página de gestão de cupons na dashboard de um restaurante. A página deve incluir as seguintes funcionalidades:

  • Tutorial de Como Funciona:
    Crie uma seção ou modal que explique como os cupons funcionam no restaurante. Isso deve ser apresentado de forma clara e concisa.
  • Aba para Cadastro de Cupons
    Implemente uma seção onde o restaurante pode cadastrar novos cupons. Os campos necessários devem incluir: nome do cupom, valor de desconto, data de validade e outras informações relevantes.
  • Listagem de Cupons:
    Crie uma seção que exiba todos os cupons cadastrados, incluindo suas informações. Os cupons devem ser organizados de forma clara e ordenados por data de validade.
  • Exclusão de Cupons:
    Permita que o restaurante exclua cupons individualmente. Isso pode ser feito com um botão "Excluir" ao lado de cada cupom na lista.
  • Gráfico de Uso dos Cupons:
    Crie um gráfico que mostre o uso dos cupons ao longo do tempo. Isso pode ser um gráfico de barras ou de pizza, dependendo do que você considerar mais adequado para a representação dos dados. (pode usar uma CDN para aplicar o gráfico)

Requisitos Técnicos

Utilize AngularJS ou Angular2+ (fica a sua escolha). Para CSS poderá ser utilizado Bootstrap. Certifique-se de que a página seja responsiva, funcionando bem em diferentes tamanhos de tela. Use boas práticas de design de interface de usuário e experiência do usuário para tornar a página intuitiva e agradável de usar, considerando que nossos clientes não tem familiaridade com tecnologia, então todo cuidado é pouco. Entrega Crie um repositório no GitHub para o projeto. Inclua um arquivo README.md que explique como executar o projeto localmente. No README.md, forneça capturas de tela e instruções de uso da página. Responda esse e-mail com o link do repositório.

Critérios de Avaliação

Seu trabalho será avaliado com base nos seguintes critérios:

  1. Qualidade do código (estrutura, legibilidade).
  2. Design de interface de usuário e experiência do usuário.
  3. Funcionalidade e completude das funcionalidades sliicitadas.
  4. Responsividade e compatibilidade com navegadores.
  5. Eficiência no uso de recursos e carregamento rápido da página.
  6. Capacidade de explicar e justificar suas decisões de design e implementação.
